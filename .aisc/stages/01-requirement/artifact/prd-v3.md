## 1. 功能目标
建立一个基于 AI 主题的视频分享平台，允许注册用户上传、发现、浏览和收藏 AI 相关视频内容，构建一个连接 AI 爱好者和创作者的社区。

## 2. 用户故事
| 角色 | 行为 | 期望结果 |
|------|------|---------|
| 注册用户 | 上传一个新的 AI 视频 | 视频成功上传，并展示在视频列表中，标题、描述和标签信息准确。 |
| 注册用户 | 在视频列表中通过关键词搜索 | 能够快速找到标题、描述或标签中包含该关键词的视频。 |
| 注册用户 | 浏览视频列表 | 能够看到按发布时间（默认）或热度排序的视频，并可以翻页查看。 |
| 注册用户 | 收藏一个喜欢的视频 | 该视频被添加到我的收藏列表中，并且收藏状态实时更新（如心形图标变红）。 |
| 注册用户 | 查看我的收藏列表 | 能够看到所有我收藏过的视频列表，并可以取消收藏。 |
| 未登录用户 | 访问平台 | 首页可浏览视频列表，但收藏按钮需提示登录或跳转登录页；直接访问上传/收藏页面会被重定向到登录页，且登录后携带redirect参数回跳。 |

## 3. 功能点
### 3.1 用户注册与登录
- **描述**：提供注册新用户和登录现有用户的功能。所有核心操作（上传、收藏）均需要身份认证。
- **输入**：
    - 注册：用户名、邮箱、密码
    - 登录：邮箱、密码
- **输出**：
    - 成功：跳转到视频列表首页或携带的redirect页面，生成并返回 JWT 令牌，并通过 Authorization Bearer 传递。
    - 失败：展示具体错误信息（如“邮箱已注册”、“密码错误”）。
- **令牌机制**：
    - 类型：JWT
    - 过期时间：24小时（Access Token），Refresh Token 有效期 7 天
    - 刷新：Access Token 过期后，前端使用 Refresh Token 通过 `/api/auth/refresh` 获取新 Access Token
    - 防范措施：
        - CSRF 防护：使用 SameSite=Strict 属性的 Cookie 传递 CSRF Token（非 HttpOnly 的 Cookie，例如 XSRF-TOKEN），前端读取该 Cookie 值并通过 `X-CSRF-Token` 请求头发送，服务端验证。同时辅以自定义请求头验证。
        - XSS 防护：对所有用户输出进行 HTML 转义，禁止直接渲染未转义的内容。
- **前置条件**：无
- **后置条件**：用户的状态变为“已登录”，系统记录用户会话。

### 3.2 视频上传
- **描述**：允许已登录用户上传视频文件，并填写元数据。
- **输入**：
    - 视频文件（支持 mp4, webm, mov 格式，暂不支持 avi）
    - 标题（必填，最长 100 字符）
    - 描述（选填，最长 500 字符）
    - 标签（选填，最多 5 个，用逗号分隔，同一视频内去重，且自动转小写并过滤空格）
- **分片上传流程**：
    1. 前端将视频文件分片（每片 5MB，最后一片可小于 5MB）。
    2. 调用 `POST /api/upload/init` 创建上传任务，返回 `taskId` 及初始上传凭证。
    3. 对每个分片，先调用 `POST /api/upload/{taskId}/signPart` 获取该分片的临时上传 URL 及签名（有效期 15 分钟），然后将分片直接上传至对象存储。
    4. 上传完成后，调用 `POST /api/upload/{taskId}/complete` 触发服务端合并文件并启动转码处理。
    5. 上传过程中若网络中断，续传时可先调用 `GET /api/upload/{taskId}/parts` 查询已成功上传的分片列表，前端跳过已上传分片，只上传缺失分片，并注意重新获取凭证。
    6. 凭证刷新策略：每个分片上传前获取新凭证；若上传过程中凭证过期，自动重新请求并重试当前分片（最多重试 3 次）。
    7. 重试策略：分片上传失败（非凭证过期）时，自动重试 3 次，间隔 1/3/5 秒；仍失败则标记该分片上传失败，整体任务进入“上传失败”状态。
- **存储方案**：采用云存储直传方式，分片直接上传至对象存储（如阿里云OSS/腾讯云COS），服务端只负责签发和验证凭证，不承担数据代理转发。视频播放时通过 CDN 分发。
- **状态查询与推送**：
    - 查询接口：`GET /api/upload/{taskId}/status` 返回当前状态及进度信息。
    - 推送机制：服务端通过 WebSocket（或者轮询，客户端基于 WebSocket 是否可用回退到 5 秒轮询）主动推送状态变更事件。
    - 状态流转：上传初始化后状态为“上传中” → 所有分片上传递交后进入“处理中”（服务端转码、截图）→ 处理完成后自动变为“已发布”或“失败”。转码失败后重试最多 2 次，若仍失败则标记为“上传失败”。
- **处理超时上限**：
    - 单文件上传总时长：从 init 到 complete 完成，超过 2 小时则自动标记为“上传失败”。
    - 转码阶段：单视频转码超时设为 30 分钟，超时后标记为“上传失败”。
- **输出**：
    - 成功：返回视频ID、最终状态（已发布/失败）。
    - 失败：返回具体错误（如“文件大小超限（最大500MB）”、“格式不支持”、“上传超时”等）。
- **前置条件**：用户已登录。
- **后置条件**：视频记录被创建，系统后台开始处理。视频出现在“我的上传”列表和公开视频列表中。

### 3.3 视频列表浏览与搜索
- **描述**：展示所有已发布的视频，支持按关键词搜索，支持排序（默认按发布时间倒序，可切换为热度排序），支持分页。
- **输入**：
    - 搜索关键词（可选，提交式触发：用户点击搜索按钮或按回车后触发搜索；**无防抖**）
    - 页码（必填，默认为 1，若超出总页数则返回最后一页内容，页码显示为最大页码）
    - 每页数量（可选，默认 20）
    - 排序方式（可选，默认 `created_at_desc`，支持 `hot_desc`）
- **输出**：
