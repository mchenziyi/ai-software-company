## 1. 功能目标
构建一个AI视频平台，核心目标是让用户能够安全、便捷地管理自己的视频内容，并发现其他用户上传的有趣视频。通过设置登录门槛，确保用户数据的私密性和安全性。该平台应具备清晰的内容上传、浏览、搜索、收藏等基础功能，为后续引入AI驱动的视频推荐、标签生成、摘要等高级功能奠定基础。

## 2. 用户故事
| 角色 | 行为 | 期望结果 |
|------|------|---------|
| 已登录用户 | 我想上传一个视频，并为其添加标题、描述和标签 | 系统成功处理我的视频，并在我上传后显示在视频列表中，且能被其他用户找到。 |
| 已登录用户 | 我想在视频列表页通过关键词搜索视频 | 系统能快速、准确地返回标题或标签中包含该关键词的视频列表。 |
| 已登录用户 | 我想翻看视频列表的不同页面 | 系统能正确展示每页的视频内容，并提供流畅的翻页体验。 |
| 已登录用户 | 我看到了一个喜欢的视频，想要收藏它 | 系统成功将该视频添加到我的个人收藏列表中，并给予明确反馈。 |
| 已登录用户 | 我想回顾我收藏过的所有视频 | 系统能完整、正确地展示我所有的收藏视频列表，并按收藏时间倒序排列。 |
| 访客（未登录） | 我想浏览或上传视频 | 系统将我引导至登录/注册页面。 |

## 3. 功能点
### 3.1 用户注册与登录
- 描述：用户可以通过用户名和密码进行注册和登录，所有核心功能都必须在登录后才能使用。
- 输入：用户名、密码、确认密码（注册时）。
- 输出：登录成功后跳转到首页（视频列表页）；注册成功后自动登录并跳转到首页；失败时返回错误提示。
- 前置条件：无。
- 后置条件：系统为该用户生成一个唯一的Session或Token，用于后续的身份验证。

### 3.2 用户上传视频
- 描述：已登录用户可以从本地选择一个视频文件，并填写标题、描述和标签（可选），然后提交上传。
- 输入：视频文件（mp4, mov, avi等常见格式）、标题（字符串，必填）、描述（文本，可选）、标签（字符串，多个用逗号分隔，可选）。
- 输出：上传成功后，系统返回上传成功的提示，视频进入后台处理（如转码）。在前端，该视频会出现在用户自己的视频列表中，或平台的视频列表里。
- 前置条件：用户已登录，文件大小和格式符合预设规则。
- 后置条件：系统启动后台任务（可以是AI任务，如自动生成标签、摘要等，本次不做强制要求，但保留接口），视频状态变为“处理中”，处理完成后变为“已发布”。

### 3.3 浏览视频列表
- 描述：用户可以浏览平台上所有用户的已发布视频列表，支持分页显示。
- 输入：页码、每页显示数量（后端预设）。
- 输出：一个包含视频信息（标题、作者、封面图、简介摘要、标签、上传时间、播放量、收藏数等）的列表。列表按上传时间倒序排列。
- 前置条件：用户已登录。
- 后置条件：无。

### 3.4 搜索视频
- 描述：用户在视频列表页顶部的搜索框中输入关键词，系统对视频的“标题”和“标签”进行模糊匹配，返回匹配的结果列表。
- 输入：搜索关键词（字符串）。
- 输出：一个包含所有匹配视频的列表，同样支持分页。如果无匹配结果，则显示“未找到相关视频”的提示。
- 前置条件：用户已登录。
- 后置条件：无。

### 3.5 收藏/取消收藏视频
- 描述：用户在浏览视频列表或视频详情页时，可以通过点击“收藏”或“取消收藏”按钮来管理自己的收藏。
- 输入：目标视频的ID。
- 输出：点击后，前端按钮状态改变（例如从空心心变为实心心），后端记录此次操作，并更新视频的收藏数。
- 前置条件：用户已登录。
- 后置条件：用户对该视频的收藏状态更新。

### 3.6 查看个人收藏列表
- 描述：用户可以进入一个专门的“我的收藏”页面，查看自己所有收藏过的视频。
- 输入：用户ID。
- 输出：一个视频列表，展示所有该用户收藏的视频信息，按收藏时间倒序排列。
- 前置条件：用户已登录。
- 后置条件：无。

## 4. 业务规则
- **认证规则**：所有核心业务功能（上传、浏览、搜索、收藏、查看收藏）均要求用户必须处于登录状态。
- **上传规则**：
  - 单个视频文件大小上限：2GB。
  - 支持的视频格式：.mp4, .mov, .avi, .mkv, .webm。
  - 视频标题长度限制：1-100个字符。
  - 视频描述长度限制：0-500个字符。
  - 标签数量限制：最多5个标签，每个标签长度限制1-20个字符。
- **收藏规则**：
  - 一个用户对同一个视频只能收藏一次。
  - 用户收藏总数无上限。
- **搜索规则**：仅支持对视频“标题”和“标签”字段进行模糊匹配搜索，不匹配视频描述。
- **内容审核**：所有上传的视频在“已发布”之前，默认需要一个“审核中”状态。在这个版本中，我们假设所有视频都自动通过审核（“无审核模式”或“自动通过”），但系统需预留“已审核”状态字段，以便未来接入AI或人工审核。
- **分页规则**：视频列表（包括首页、搜索结果、收藏列表）每页固定显示20个视频。
- **排序规则**：平台视频列表默认按上传时间降序排列；个人收藏列表默认按收藏时间降序排列。

## 5. 边界条件
| 场景 | 预期行为 |
|------|---------|
| 空数据 | 无视频时，视频列表页显示“暂无视频，快来上传第一个视频吧！”；无收藏时，收藏页显示“您还没有收藏视频”。 |
| 极限值 | 上传一个正好2GB的视频：上传成功；上传一个2.1GB的视频：上传失败，提示“文件大小超出限制”。 |
| 并发 | 多个用户同时上传视频：系统正常处理，每个用户彼此不受影响，视频在列表中按上传时间正确排序。 |
| 异常输入 | 搜索框中输入`<script>alert('xss')</script>`：系统应对输入进行转义处理，防止XSS攻击，并正常返回无结果的搜索结果。 |
| 重复收藏 | 用户收藏一个已经收藏过的视频：系统提示“您已收藏过该视频”，收藏状态不变。 |
| 取消收藏空列表 | 用户在收藏列表页，取消了列表中最后一个视频的收藏：系统正常取消，列表变为空，并显示“您还没有收藏视频”。 |

## 6. 验收标准
- [ ] **用户上传**：登录后，能成功上传一个符合格式和大小的视频，填写标题、描述、标签，上传后能在平台上搜索到。
- [ ] **视频列表**：登录后，平台首页正确展示所有已发布的视频列表，并支持点击页码进行翻页。
- [ ] **搜索功能**：登录后，在搜索框输入关键词，能正确搜索出标题或标签包含该关键词的视频列表；输入空白或无效关键词，能正确显示无结果。
- [ ] **收藏功能**：登录后，在视频列表或详情页点击收藏按钮，按钮状态改变，该视频出现在用户的“我的收藏”列表中。再次点击可取消收藏。
- [ ] **鉴权**：未登录状态下，访问任何功能页面（如列表、上传、收藏）时，被自动重定向到登录页面。
- [ ] **边界条件**：尝试上传超过2GB的文件，系统拒绝并给出提示；无视频时，首页显示空状态提示；收藏列表为空时，显示空状态提示。

## 7. 不做什么（Out of Scope）
- 视频播放、转码、截图功能（本版本仅做上传和列表展示/收藏，不包含视频播放页面的交互逻辑）。
- AI特性（如自动标签生成、摘要、推荐算法、视频内容分析、语音转文字等）。
- 视频编辑（如剪切、添加字幕、滤镜等）。
- 用户个人资料编辑（如修改头像、昵称、密码）。
- 视频评论、点赞、分享功能。
- 视频观看历史记录。
- 用户消息通知系统。
- 移动端原生App或小程序（仅考虑Web端）。
- 后台管理界面（用于管理员审核、管理用户和视频）。
- 视频打赏或付费功能。

## 8. 待澄清问题
[NEEDS CLARIFICATION]
1. **视频播放器**：虽然播放不是当前版本的核心功能，但列表中的视频需要展示。用户点击视频后，是期望直接跳转到一个播放页面，还是点击后立即在原地播放？我们假设本版本暂不实现播放，仅展示视频的静态封面图（系统自动截取第一帧或用户上传封面）。
2. **“AI视频平台”的AI属性**：当前需求中未体现AI。这是否意味着此版本为纯粹的“基础版视频平台”，AI特性是后续版本（V2.0）的重点？如果是，需要确认V1.0的目标就是完成上述基础功能。
3. **用户注册/登录方式**：除用户名密码外，是否支持第三方登录（如微信、Google、GitHub）？本版本假设仅支持基本的用户名密码注册登录。
4. **视频封面**：上传视频时，用户是否有权限上传自定义封面图？还是自动截取视频第一帧作为封面？假设为自动截取第一帧（这个功能在后台处理时实现，但不在本次需求范围内）。
5. **视频状态**：上传后到“审核/处理完成”需要时间，在此期间视频在列表中是显示为“处理中”还是完全不可见？假设是“处理中”状态对用户不可见，仅供上传者自己在个人管理页面查看。
