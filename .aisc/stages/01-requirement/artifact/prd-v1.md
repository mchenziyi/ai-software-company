## 1. 功能目标
建立一个基于 AI 主题的视频分享平台，允许注册用户上传、发现、浏览和收藏 AI 相关视频内容，构建一个连接 AI 爱好者和创作者的社区。

## 2. 用户故事
| 角色 | 行为 | 期望结果 |
|------|------|---------|
| 注册用户 | 上传一个新的 AI 视频 | 视频成功上传，并展示在视频列表中，标题、描述和标签信息准确。 |
| 注册用户 | 在视频列表中通过关键词搜索 | 能够快速找到标题、描述或标签中包含该关键词的视频。 |
| 注册用户 | 浏览视频列表 | 能够看到按发布时间或热度排序的视频，并可以翻页查看。 |
| 注册用户 | 收藏一个喜欢的视频 | 该视频被添加到我的收藏列表中，并且收藏状态实时更新（如心形图标变红）。 |
| 注册用户 | 查看我的收藏列表 | 能够看到所有我收藏过的视频列表，并可以取消收藏。 |
| 未登录用户 | 访问平台 | 看到的应该是登录/注册页面，或仅能浏览视频列表但无法上传或收藏。 |

## 3. 功能点
### 3.1 用户注册与登录
- **描述**：提供注册新用户和登录现有用户的功能。所有核心操作（上传、收藏）均需要身份认证。
- **输入**：
    - 注册：用户名、邮箱、密码
    - 登录：邮箱、密码
- **输出**：
    - 成功：跳转到视频列表首页，生成并返回认证令牌（Token）。
    - 失败：展示具体错误信息（如“邮箱已注册”、“密码错误”）。
- **前置条件**：无
- **后置条件**：用户的状态变为“已登录”，系统记录用户会话。

### 3.2 视频上传
- **描述**：允许已登录用户上传视频文件，并填写元数据。
- **输入**：
    - 视频文件 (支持 mp4, webm 等常见格式)
    - 标题 (必填，最长 100 字符)
    - 描述 (选填，最长 500 字符)
    - 标签 (选填，支持多个，用逗号分隔)
- **输出**：
    - 成功：返回视频上传成功信息，视频状态为“处理中”或“已发布”。
    - 失败：返回具体错误（如“文件大小超限”、“格式不支持”）。
- **前置条件**：用户已登录。
- **后置条件**：视频记录被创建，系统后台开始处理（如转码、截图等）。视频出现在“我的上传”列表和公开视频列表中。

### 3.3 视频列表浏览与搜索
- **描述**：展示所有已发布的视频，支持按关键词搜索，并支持分页。
- **输入**：
    - 搜索关键词 (可选)
    - 页码 (必填，默认为 1)
    - 每页数量 (可选，默认 20)
- **输出**：视频信息列表，包含标题、描述片段、上传者、缩略图、标签、播放量等。还需返回当前页码、总页数和总视频数。
- **前置条件**：无 (未登录用户也可访问)。
- **后置条件**：无。

### 3.4 视频收藏
- **描述**：允许已登录用户收藏一个视频，或取消收藏。
- **输入**：
    - 用户ID (从会话中获取)
    - 视频ID
    - 操作类型 (收藏 / 取消收藏)
- **输出**：
    - 成功：更新后的收藏状态 (true/false) 和当前视频的收藏总数。
- **前置条件**：用户已登录。
- **后置条件**：用户的收藏列表更新。视频的收藏数增加或减少。

### 3.5 我的收藏列表
- **描述**：展示当前登录用户所有收藏过的视频。
- **输入**：
    - 用户ID (从会话中获取)
    - 页码
- **输出**：当前登录用户的所有收藏视频列表，支持分页。每个视频项需包含与视频列表页类似的信息，并显示“已收藏”状态，方便取消。
- **前置条件**：用户已登录。
- **后置条件**：无。

## 4. 业务规则
- **认证要求**：未登录用户可以浏览视频列表（功能 3.3），但不能上传（功能 3.2）或收藏（功能 3.4、3.5）。执行这些操作时必须重定向到登录页。
- **内容审核**：上传的视频在首次发布前需要经过简单审核（标注为“审核中”），审核通过后才能在列表中被所有人看到。初始 MVP 可设定为自动通过。
- **标签规范化**：系统应提取并规范化标签（如转小写、去空格），避免相同含义的标签重复出现。
- **唯一性**：同一用户不能收藏同一个视频两次。再次点击收藏为取消收藏（Toggle 操作）。
- **用户数据隔离**：用户只能看到和操作自己的收藏列表。

## 5. 边界条件
| 场景 | 预期行为 |
|------|---------|
| 空数据 | **视频列表页**: 展示空状态提示，如“还没有 AI 视频，快来上传第一个吧！”<br>**搜索无结果**: 展示“没有找到相关视频”的提示。<br>**收藏列表空**: 展示“你还没有收藏任何视频，去探索一下吧！”提示。 |
| 极限值 | **视频大小**: 假设设定单个视频最大 500MB，如果超限，上传失败并给出明确提示。<br>**标题长度**: 输入超过 100 字符时，输入框禁止继续输入或提交时截断并提示。<br>**页码**: 访问超过总页数的页码（如总页数为10，访问第11页），返回最后一页内容。 |
| 并发 | **收藏操作**: 同一用户快速双击收藏/取消收藏按钮，系统应正确处理，只产生一次有效操作（接口需要支持幂等性或在客户端做防抖处理）。<br>**视频上传**: 多用户同时上传，系统能正常排队处理，不阻塞。 |
| 异常输入 | **文件格式**: 上传非视频文件（如 .txt, .exe），系统拒绝并提示“仅支持 mp4, webm, avi 等视频格式”。<br>**搜索**: 输入特殊字符（如 SQL 注入语句）进行搜索，系统应做转义处理，返回正常搜索结果或空结果。 |

## 6. 验收标准
- [ ] **用户注册功能**：用户能使用邮箱和密码成功注册，注册后自动登录并跳转到首页。重复邮箱注册会失败并提示。
- [ ] **用户登录功能**：用户能使用注册的邮箱和密码成功登录。密码错误会提示。
- [ ] **视频上传功能**：登录用户可以成功上传一个不超过 500MB 的 .mp4 文件，并填写标题和标签。上传成功后，该视频出现在视频列表中。
- [ ] **视频列表分页**：当视频超过 20 个时，列表下方显示分页控件，点击第 2 页能正确加载并显示第 21-40 个视频。
- [ ] **视频搜索功能**：在搜索框输入“机器学习”，搜索结果应包含所有标题、描述或标签中包含“机器学习”的视频。
- [ ] **视频收藏功能**：登录用户在浏览视频详情页或列表页时，点击收藏按钮，按钮状态变为“已收藏”。再次点击，状态恢复为“未收藏”。
- [ ] **我的收藏列表**：登录用户点击导航栏中的“我的收藏”，页面展示该用户所有已收藏的视频列表。
- [ ] **未登录访问**：未登录用户尝试访问上传页面或收藏页面，会被重定向到登录页面。

## 7. 不做什么（Out of Scope）
- 视频播放页面（点击后跳转到第三方或仅提供播放/暂停功能，不做弹幕、评论、点赞等社交功能）。
- 视频内容推荐算法（如“猜你喜欢”）。
- 用户个人主页（上传视频列表、信息编辑等）。
- 后台管理系统（仅限普通用户端功能）。
- 视频转码或压缩能力（假设由第三方服务处理）。
- 通知系统（如注册成功邮件、收藏通知）。
