## 1. 功能目标
建立一个基于 AI 主题的视频分享平台，允许注册用户上传、发现、浏览和收藏 AI 相关视频内容，构建一个连接 AI 爱好者和创作者的社区。

## 2. 用户故事
| 角色 | 行为 | 期望结果 |
|------|------|---------|
| 注册用户 | 上传一个新的 AI 视频 | 视频成功上传，并展示在视频列表中，标题、描述和标签信息准确。 |
| 注册用户 | 在视频列表中通过关键词搜索 | 能够快速找到标题、描述或标签中包含该关键词的视频。 |
| 注册用户 | 浏览视频列表 | 能够看到按发布时间（默认）或热度排序的视频，并可以翻页查看。 |
| 注册用户 | 收藏一个喜欢的视频 | 该视频被添加到我的收藏列表中，并且收藏状态实时更新（如心形图标变红）。 |
| 注册用户 | 查看我的收藏列表 | 能够看到所有我收藏过的视频列表，并可以取消收藏。 |
| 未登录用户 | 访问平台 | 首页可浏览视频列表，但收藏按钮需提示登录或跳转登录页；直接访问上传/收藏页面会被重定向到登录页，且登录后携带redirect参数回跳。 |

## 3. 功能点
### 3.1 用户注册与登录
- **描述**：提供注册新用户和登录现有用户的功能。所有核心操作（上传、收藏）均需要身份认证。
- **输入**：
    - 注册：用户名、邮箱、密码
    - 登录：邮箱、密码
- **输出**：
    - 成功：跳转到视频列表首页或携带的redirect页面，生成并返回 JWT 令牌，并通过 Authorization Bearer 传递。
    - 失败：展示具体错误信息（如“邮箱已注册”、“密码错误”）。
- **令牌机制**：
    - 类型：JWT
    - 过期时间：24小时（Access Token），Refresh Token 有效期 7 天
    - 刷新：Access Token 过期后，前端使用 Refresh Token 通过 `/api/auth/refresh` 获取新 Access Token
    - 防范措施：所有接口要求 CSRFToken（通过 HTTP-only Cookie 传递），且对用户输入进行 HTML 转义防止 XSS
- **前置条件**：无
- **后置条件**：用户的状态变为“已登录”，系统记录用户会话。

### 3.2 视频上传
- **描述**：允许已登录用户上传视频文件，并填写元数据。
- **输入**：
    - 视频文件（支持 mp4, webm, mov 格式，暂不支持 avi）
    - 标题（必填，最长 100 字符）
    - 描述（选填，最长 500 字符）
    - 标签（选填，最多 5 个，用逗号分隔，同一视频内去重，且自动转小写并过滤空格）
- **分片支持**：前端将视频文件分片（每片 5MB），通过 `POST /api/upload/init` 创建上传任务，后续用 `PUT /api/upload/{taskId}/part` 逐片上传递，上传完成后调用 `POST /api/upload/{taskId}/complete` 触发服务端合并。
- **存储方案**：采用云存储直传方式，分片直接上传至对象存储（如阿里云OSS/腾讯云COS），服务端签发临时上传凭证。视频播放时通过 CDN 分发。
- **状态流转**：上传初始化后状态为“上传中” → 所有分片上传递交后进入“处理中”（服务端转码、截图）→ 处理完成后自动变为“已发布”或“失败”。用户可在上传列表实时查看状态。
- **输出**：
    - 成功：返回视频ID、最终状态（已发布/失败）。
    - 失败：返回具体错误（如“文件大小超限（最大500MB）”、“格式不支持”）。
- **前置条件**：用户已登录。
- **后置条件**：视频记录被创建，系统后台开始处理。视频出现在“我的上传”列表和公开视频列表中。

### 3.3 视频列表浏览与搜索
- **描述**：展示所有已发布的视频，支持按关键词搜索，支持排序（默认按发布时间倒序，可切换为热度排序），支持分页。
- **输入**：
    - 搜索关键词（可选，提交式触发，非实时搜索）
    - 页码（必填，默认为 1）
    - 每页数量（可选，默认 20）
    - 排序方式（可选，默认 `created_at_desc`，支持 `hot_desc`）
- **输出**：视频信息列表，包含标题、描述片段、上传者、缩略图、标签、播放量等。还需返回当前页码、总页数和总视频数。
- **搜索方案**：采用 Elasticsearch 作为搜索引擎，MySQL 作为数据主库。同步机制为：视频发布时通过事件机制异步写入 ES（延迟≤5秒）。ES 索引字段包括标题（ik_smart分词）、描述（ik_smart分词）、标签（keyword类型）。性能要求：单次搜索响应时间＜300ms，支持 1000 QPS。
- **前端规范**：用户点击搜索按钮或按回车后触发搜索，输入框添加 300ms 防抖（避免无效请求），分页状态保留（翻页时保持搜索关键词和排序方式）。
- **前置条件**：无（未登录用户也可访问）。
- **后置条件**：无。

### 3.4 视频收藏（Toggle操作）
- **描述**：允许已登录用户收藏一个视频，再次点击取消收藏。
- **输入**：
    - 用户ID（从会话中获取）
    - 视频ID
    - 不传操作类型，由后端根据是否存在记录决定（Toggle 语义）
- **输出**：
    - 成功：返回新的收藏状态（true/false）和当前视频的收藏总数。
- **后端幂等**：用户-视频收藏表建立唯一索引（user_id, video_id），保证同一用户不能重复收藏同一视频。接口设计为：先查询是否存在，若存在则删除记录并返回 false；若不存在则插入记录并返回 true。
- **前置条件**：用户已登录。
- **后置条件**：用户的收藏列表更新。视频的收藏数增加或减少。

### 3.5 我的收藏列表
- **描述**：展示当前登录用户所有收藏过的视频。
- **输入**：
    - 用户ID（从会话中获取）
    - 页码
- **输出**：当前登录用户的所有收藏视频列表，支持分页。每个视频项需包含与视频列表页类似的信息，并显示“已收藏”状态，方便取消。
- **前置条件**：用户已登录。
- **后置条件**：无。

## 4. 业务规则
- **认证要求**：未登录用户可以浏览视频列表（功能 3.3），但收藏按钮需显示“请先登录”，点击后跳转登录页并带 redirect 参数；访问上传页面或收藏页面时直接重定向到登录页并携带 redirect 参数，登录成功后回跳。
- **内容审核**：
    - MVP 阶段：上传视频自动通过审核（状态直接变为“已发布”），但系统记录所有视频的元数据以备后续追溯。需考虑风险：若出现违规内容，需提供举报入口（本版本暂不实现，但预留接口）。
    - 后续迭代：引入人工审核 + AI 辅助审核（图片/视频内容识别），视频发布前状态为“审核中”，审核通过后变为“已发布”。
- **标签规范化**：
    - 最大数量：每个视频最多 5 个标签。
    - 去重：同一视频内标签不可重复。
    - 处理：前端提交时自动转小写、去除首尾空格、根据逗号分割后去重，服务端再次校验。
- **唯一性**：同一用户不能收藏同一个视频两次。再次点击收藏为取消收藏（Toggle 操作）。
- **用户数据隔离**：用户只能看到和操作自己的收藏列表。

## 5. 全局加载与错误处理
- **加载状态**：所有涉及网络请求的场景（列表加载、搜索、上传进度、登录注册等）均需显示 Skeleton（卡片/列表占位）或 Spinner（按钮加载）。特定操作（如收藏）可仅使用按钮内 Spinner。
- **错误处理**：
    - 网络中断：全局捕获 fetch 错误，显示 Toast 提示“网络连接失败，请检查网络”，并提供重试按钮。
    - 请求超时：统一超时时间设定为 10 秒，超时后显示“请求超时，请稍后重试”。
    - 服务端错误（5xx）：显示“服务器繁忙，请稍后重试”。
    - 业务错误（4xx）：展示对应错误提示（如 401 跳转登录页，403 提示无权限）。

## 6. 边界条件
| 场景 | 预期行为 |
|------|---------|
| 空数据 | **视频列表页**: 展示空状态提示，如“还没有 AI 视频，快来上传第一个吧！”<br>**搜索无结果**: 展示“没有找到相关视频”的提示。<br>**收藏列表空**: 展示“你还没有收藏任何视频，去探索一下吧！”提示。 |
| 极限值 | **视频大小**: 假设设定单个视频最大 500MB，如果超限，上传失败并给出明确提示（前端预检）。<br>**标题长度**: 输入超过 100 字符时，输入框禁止继续输入或提交时截断并提示。<br>**页码**: 访问超过总页数的页码（如总页数为10，访问第11页），返回最后一页内容，页码显示为最大页码。<br>**标签数量**: 输入超过 5 个标签，前端阻止提交并提示“最多 5 个标签”。 |
| 并发 | **收藏操作**: 同一用户快速双击收藏/取消收藏按钮，后端通过唯一索引保证幂等，前端可防抖（300ms）减少无效请求。<br>**视频上传**: 多用户同时上传，系统能正常排队处理，不阻塞，云存储直传不受影响。 |
| 异常输入 | **文件格式**: 上传非视频文件（如 .txt, .exe），系统拒绝并提示“仅支持 mp4, webm, mov 等视频格式”。<br>**搜索**: 输入特殊字符（如 SQL 注入语句）进行搜索，系统做转义处理，Elasticsearch 也会处理特殊字符，返回正常搜索结果或空结果。 |

## 7. 验收标准
- [ ] **用户注册功能**：用户能使用邮箱和密码成功注册，注册后自动登录并跳转到首页或 redirect 页面。重复邮箱注册会失败并提示“该邮箱已注册”。
- [ ] **用户登录功能**：用户能使用注册的邮箱和密码成功登录。密码错误时提示“密码错误，请重新输入”。连续 5 次登录失败需验证码（本版本暂缓，但记录失败次数）。
- [ ] **视频上传功能**：
    - 登录用户可以成功上传一个不超过 500MB 的 .mp4 文件，并填写标题和标签。
    - 上传失败场景：网络中断后文件上传进度暂停，可续传；格式不支持（如 .avi）提示错误；文件过大提示错误。
    - 上传完成后视频状态流转为“上传中”→“处理中”→“已发布”，在“我的上传”列表中可查看状态。
- [ ] **视频列表分页**：当视频超过 20 个时，列表下方显示分页控件，点击第 2 页能正确加载并显示第 21-40 个视频。访问超过总页数的页码（如第 99 页），应返回最后一页内容，且页码显示为最大页码。
- [ ] **视频搜索功能**：在搜索框输入“机器学习”，点击搜索按钮或按回车，搜索结果应包含所有标题、描述或标签中包含“机器学习”的视频（支持分词）。搜索后分页、排序状态保留。搜索无结果时显示空状态。
- [ ] **视频收藏功能**：
    - 登录用户在浏览视频列表页或详情页时，点击收藏按钮，按钮状态变为“已收藏”，收藏数+1。
    - 再次点击同一按钮，状态恢复为“未收藏”，收藏数-1（取消收藏验证）。
    - 未登录用户点击收藏按钮，弹出提示或跳转登录页，登录后回跳并自动执行收藏（可选，本版本仅跳转）。
- [ ] **我的收藏列表**：登录用户点击导航栏中的“我的收藏”，页面展示该用户所有已收藏的视频列表。收藏列表为空时显示空状态提示。
- [ ] **未登录访问**：未登录用户直接访问 `/upload` 或 `/favorites` 页面，会被重定向到 `/login?redirect=/upload`（或 `/favorites`），登录成功后自动跳转回原目标页面。
- [ ] **全局加载与错误**：所有页面首次加载显示 Skeleton，网络断开时弹出 Toast 提示并允许重试，超时显示“请求超时”。

## 8. 不做什么（Out of Scope）
- 视频播放页面（点击后跳转到第三方或仅提供播放/暂停功能，不做弹幕、评论、点赞等社交功能）。
- 视频内容推荐算法（如“猜你喜欢”）。
- 用户个人主页（上传
